# MABCO - Mobile & Electronics E-Commerce Platform

## 🚀 Overview

MABCO is a modern, feature-rich e-commerce platform specializing in mobile devices, accessories, and electronics. Built with a powerful React frontend and Node.js/Express backend, MABCO provides customers with a seamless shopping experience while offering comprehensive management tools for business operations.

**Platform**: Full-stack web application with dedicated admin dashboard  
**Supported Languages**: Arabic & English (fully bilingual)  
**Target Market**: Middle East region

## 🏗️ Technology Stack

### Frontend
- **Framework**: React 18 with TypeScript
- **Build Tool**: Vite
- **Styling**: Tailwind CSS with Radix UI components
- **Routing**: React Router v7
- **State Management**: Context API, Zustand
- **Form Handling**: React Hook Form
- **Charts & Data**: Recharts, React CountUp
- **Animations**: Motion (Framer Motion alternative)
- **Carousel**: Embla Carousel
- **Internationalization**: i18n support
- **Authentication**: Firebase
- **UI Components**: Extensive Radix UI library

### Backend
- **Runtime**: Node.js
- **Framework**: Express.js
- **Database**: MongoDB
- **Authentication**: Firebase Admin SDK
- **Validation**: Joi
- **File Upload**: Multer
- **Security**: Helmet, CORS, Rate Limiting
- **Additional DB**: MSSQL support
- **Development**: Nodemon for hot reload

## 🚀 Getting Started

### Prerequisites
- Node.js (v18 or higher)
- MongoDB instance
- Firebase project setup
- Environment variables configured

### Installation & Setup

#### 1. Frontend Setup
```bash
cd client
npm install
npm run dev  # Start development server (localhost:5173)
```

#### 2. Backend Setup
```bash
cd server
npm install
cp .env.example .env  # Configure environment variables
npm run db:init       # Initialize MongoDB schema
npm run dev           # Start backend (localhost:5000)
```

### Build for Production
```bash
# Frontend
cd client
npm run build

# Backend is ready with npm start
cd server
npm start
```

## 📊 Main API Endpoints

### Public Endpoints
- `GET /api/health` - Health check
- `GET /api/products` - List all products
- `GET /api/products/:id` - Get product details
- `GET /api/categories` - List categories
- `GET /api/brands` - List brands
- `GET /api/offers` - Get active offers
- `GET /api/showrooms` - Get showroom locations
- `GET /api/faqs` - Get FAQ items

### Customer Endpoints
- `GET /api/orders` - User order history
- `POST /api/orders` - Create new order
- `GET /api/cart` - Get shopping cart
- `POST /api/cart` - Add to cart
- `POST /api/notifications` - Subscribe to notifications

### Admin Endpoints (Requires API Key)
- `GET /api/admin/reports/daily-kpis` - Dashboard KPIs
- `GET /api/admin/actions` - Admin action logs
- `GET /api/admin/product-revisions/:productId` - Product change history
- `GET /api/admin/users/:id/permissions` - User permissions
- `PUT /api/admin/users/:id/permissions` - Update user permissions
- Full CRUD operations for products, offers, orders, users, etc.
